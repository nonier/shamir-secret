
Функционал работает через REST запросы на локальный сервер
Приложение запускается на 80 порту

Добавлена документация swagger, для проверки эндпоинтов можно перейти
на http://localhost:8080/swagger-ui.html или воспользоваться другими утилитами,
например Postman.

Поддерживаемые методы:

POST: /secret
Создает и разбивает секрет на части
В параметрах запроса передается количество частей для генерации в виде:
    partsCount=4
Как тело запроса передается JSON в виде:

{
    "secret": 12345,        //значение секрета
    "p": 4                  //поле, в котором будут создаваться ключи
}

GET: /secret
Пытается восстановить секрет из входных частей
Как тело запроса передается JSON в виде:
{
[
    {
    "point": 12,            //значение x
    "value": 2345,          //значение y
    "p": 4                  //поле, в котором создался ключ
    },
    {
    "point": 33,
    "value": 4212,
    "p": 4
    }
]
}

PUT: /secret
Пытается сгенерировать новые части для секрета
В параметрах запроса передается количество частей для генерации в виде:
    partsCount=4
Как тело запроса передается JSON в виде:
{
[
    {
    "point": 12,            //значение x
    "value": 2345,          //значение y
    "p": 4                  //поле, в котором создался ключ
    },
    {
    "point": 33,
    "value": 4212,
    "p": 4
    }
]
}
